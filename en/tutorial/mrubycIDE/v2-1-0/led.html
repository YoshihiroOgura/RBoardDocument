RBoard

言語: 🇯🇵 日本語 / 🇺🇸 English

First, install the IDE!

Please refer to the mruby/cIDE installation page for instructions.

Writing the Program

  • First, launch the mruby/cIDE (mrubyc_ide.exe)
  • Select "File -> New… -> Project"
  • Then the "Setup New Project" tab will appear.
  • You must enter a Project Name (name it "led")
  • Click the OK button to close
  • Select "File -> New… -> File"
  • Then the "Setup Add New File" tab will appear.
  • You need to enter the file name, so enter it.
    • The file extension must be [.rb] (name it main.rb)
  • Write the program. Repeat LED ON/OFF every 1 second.
    • 
while  true # Infinite loop
  digitalWrite(0, 1) # LED1 ON
  sleep(1) # 1 second sleep
  digitalWrite(0, 0) # LED1 OFF
  sleep(1) # 1 second sleep
end
    The format is digitalWrite(pin number, output value), and for the pin number, you specify Pin0 to Pin20 (0 to 20) as listed in the Pin Configuration etc. section for surface mount components.
    Each of the four LEDs is connected to Pin0, Pin1, Pin5, and Pin6.
    The output value for the second argument is 1 for ON (3.3V) and 0 for OFF (0V) for the pin number specified by the first argument.

    Writing the Program to RBoard

  • After writing, connect the RBoard to the computer (using a USB-microUSB cable)
    • ※Use a data transfer cable for the USB cable. (Charging cables will not work)
  • Select [File -> Settings…]
  • Check if the Port setting under [Build and Deploy -> Writer] is set to the RBoard port.
    • In the case below, there are COM12 and COM19, and COM12 is RBoard.
  • Click the OK button to close.
  • Click the Write button in the IDE to start the writing process.
  • Press the RS button on the RBoard immediately after [Start connection.] is displayed.
    • Or, unplug and replug the USB cable connecting the RBoard and PC. (The latter method is actually more reliable)

    Log in case of failure
    If the writing process fails, try unplugging and replugging the USB cable. 
    
Start connection.
..........
                  
Can't connect target device.
    If the COM port to which the RBoard is connected is different, check the port in [Build and Deploy -> Writer]. 
    
Start connection.
Can't open serial port line.
  • The process is complete when LED1 (D1) starts blinking.

Task:

  • Try changing the blinking speed of the LED.
    • 
while  true # Infinite loop
  digitalWrite(0, 1) # LED1 ON
  sleep(0.1) # 0.1 second sleep
  digitalWrite(0, 0) # LED1 OFF
  sleep(0.1) # 0.1 second sleep
end
  • Try blinking two LEDs alternately.
    • 
while  true # Infinite loop
  digitalWrite(0, 1) # LED1 ON
  sleep(1) # 1 second sleep
  digitalWrite(1, 1) # LED2 ON
  digitalWrite(0, 0) # LED1 OFF
  sleep(1) # 1 second sleep
  digitalWrite(1, 0) # LED2 OFF
end
  • Try blinking four LEDs in sequence.
    • 
leds = [0, 1, 5, 6]
while  true
  leds.each do |i|
    digitalWrite(i,1)
    sleep(1)
    digitalWrite(i,0)
  end
end
